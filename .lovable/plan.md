

# Deep Search для HikkoGPT

## Обзор

Добавляем функцию "Глубокий поиск" -- многошаговый процесс, который:
1. Задает пользователю уточняющие вопросы (пронумерованные)
2. Генерирует поисковые запросы через AI-планировщик
3. Ищет и скрапит источники через Firecrawl
4. Синтезирует финальный отчет через AI-аналитик

Лимит: 1 глубокий поиск на чат.

---

## Шаг 0: Подключение Firecrawl

Firecrawl нужен для веб-поиска и скрапинга. Его `search` endpoint умеет и искать, и возвращать контент страниц -- это упрощает архитектуру (не нужен отдельный Serper API).

Подключим коннектор Firecrawl к проекту через настройки.

---

## Шаг 1: Edge Function `deepsearch`

Создаем `supabase/functions/deepsearch/index.ts` -- основной бэкенд для глубокого поиска.

**Этапы внутри функции:**

```text
+-----------------------+
| 1. Получить запрос    |
|    + контекст от юзера|
+-----------------------+
         |
         v
+-----------------------+
| 2. AI-Планировщик     |
|    (gemini-3-flash)   |
|    -> уточняющие      |
|       вопросы ИЛИ     |
|    -> поисковые       |
|       запросы         |
+-----------------------+
         |
         v
+-----------------------+
| 3. Firecrawl Search   |
|    (10-20 запросов    |
|     параллельно)      |
+-----------------------+
         |
         v
+-----------------------+
| 4. Фильтрация +      |
|    дедубликация URLs  |
+-----------------------+
         |
         v
+-----------------------+
| 5. AI-Аналитик        |
|    (gemini-3-pro)     |
|    -> финальный отчет |
+-----------------------+
```

**Логика работы:**

Функция принимает `action` параметр:
- `action: "clarify"` -- AI генерирует 3-5 уточняющих вопросов по запросу пользователя (пронумерованных)
- `action: "search"` -- запускает полный цикл поиска с учетом ответов на уточняющие вопросы

**Фаза поиска:**
1. AI-планировщик (gemini-3-flash-preview) генерирует 10-20 поисковых запросов через tool calling
2. Параллельные вызовы Firecrawl Search API (с `scrapeOptions` для получения markdown)
3. Агрегация результатов, дедубликация по URL
4. Очистка и обрезка текста (лимит ~100K токенов контекста)
5. AI-аналитик (gemini-3-pro-preview) синтезирует структурированный отчет с источниками

## Шаг 2: UI компоненты

### 2.1 Кнопка "Глубокий поиск"

Добавляем кнопку в `ChatInput.tsx` рядом с кнопкой прикрепления изображения. Иконка -- `Search` из lucide-react.

### 2.2 Режим Deep Search в чате

Когда пользователь нажимает кнопку и вводит запрос:
1. Отправляется запрос на `clarify`
2. Бот показывает пронумерованные уточняющие вопросы
3. Пользователь отвечает на вопросы
4. Бот запускает полный поиск, показывая прогресс:
   - "Генерирую поисковые запросы..."
   - "Ищу информацию (найдено X источников)..."
   - "Анализирую источники..."
   - Финальный отчет

### 2.3 Лимит

В `useChat.ts` добавляем трекинг: проверяем, был ли уже deep search в текущем чате. Если да -- кнопка становится неактивной с подсказкой "Лимит исчерпан".

## Технические детали

### Edge Function -- фаза clarify

Вызывает Lovable AI с tool calling:
```text
Tool: generate_clarifying_questions
Parameters: questions[] -- массив из 3-5 вопросов с номерами
```

### Edge Function -- фаза search

1. Tool calling для генерации запросов:
```text
Tool: generate_search_queries  
Parameters: queries[] -- массив из 10-20 строк
```

2. Параллельный Firecrawl search (по 5 запросов одновременно, чтобы не превысить лимиты)

3. Финальный анализ -- обычный streaming completion через gemini-3-pro-preview

### Структура ответа deep search

Отчет в формате Markdown:
- Заголовок и краткое резюме
- Основные находки (структурированные секции)
- Источники (нумерованный список с URL)
